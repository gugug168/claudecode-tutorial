## 为什么要记录会话？

### 问题：开发中的遗忘

你是否遇到过：
- 第二天继续工作，忘记昨天做到哪了
- 遇到过的问题又遇到了，忘记怎么解决的
- 临时切换任务，回来后不知道从哪里继续

### 解决方案：会话记录

通过记录会话，你可以：
- **快速恢复** - 读取会话记录，立刻知道当前状态
- **知识积累** - 解决过的问题成为可复用的经验
- **团队协作** - 其他人可以接手你的工作

## 会话记录结构

### 标准模板

```markdown
# Session: 会话标题
**Date:** 日期
**Started:** 开始时间
**Last Updated:** 最后更新时间

---

## Current State
当前状态的简要描述

### Completed
- [x] 已完成项 1
- [x] 已完成项 2

### In Progress
- [ ] 进行中项 1
- [ ] 进行中项 2

### Blockers Encountered
遇到的障碍及解决方案

### Key Decisions Made
关键决策及原因

### Notes for Next Session
下次会话需要注意的事项

### Context to Load
需要加载的文件路径
```

## 三种会话类型

### 1. 调试类会话

**特点**：排查问题、寻找根因

**记录重点**：
- 问题描述（什么现象）
- 调试过程（尝试了什么）
- 根本原因（为什么发生）
- 修复方案（怎么解决的）
- 调试技巧（可复用的方法）

### 2. 重构类会话

**特点**：改进代码结构、统一规范

**记录重点**：
- 重构目标
- 当前进度（已完成/待完成）
- 发现的问题
- 确定的标准/规范
- 下一步计划

### 3. 功能开发类会话

**特点**：实现新功能

**记录重点**：
- 功能目标
- 技术选型及原因
- 遇到的障碍和解决方法
- 代码修改位置
- 时间线日志（可选）

## 最佳实践

### 1. 会话开始时

```markdown
# 创建新会话文件
**Date:** 2026-01-20
**Started:** 09:00

## Current State
今天要做什么...
```

### 2. 会话进行中

```markdown
# 及时更新进度
### Completed
- [x] 新完成的项目

### Blockers Encountered
1. 遇到的问题及解决方法
```

### 3. 会话结束时

```markdown
# 保存状态
### Notes for Next Session
- 明天继续做什么
- 需要注意什么

### Context to Load
下次需要打开的文件
```
