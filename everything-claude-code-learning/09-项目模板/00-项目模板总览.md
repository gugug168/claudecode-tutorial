## 如何使用这些模板

### 第一步：选择合适的模板

根据你的项目类型选择：
- **做网站/前端应用** → 基础项目模板 或 SaaS 模板
- **做 Python 后端** → Django API 模板
- **做 Go 后端** → Go 微服务模板
- **做 Rust 后端** → Rust API 模板

### 第二步：复制到项目根目录

```bash
# 将模板复制到你的项目根目录，命名为 CLAUDE.md
cp 模板路径/xx-项目模板.md 你的项目/CLAUDE.md
```

### 第三步：根据项目修改

模板中需要修改的部分：
1. **项目概述** - 描述你的项目做什么
2. **环境变量** - 填入你实际使用的变量
3. **文件结构** - 调整为你的实际目录结构
4. **特殊规则** - 添加你项目特有的规则

---

## 模板结构说明

每个模板都包含以下部分：

### 1. 项目概述
```markdown
## Project Overview
技术栈、架构说明
```

### 2. 关键规则
```markdown
## Critical Rules
- 代码组织规则
- 数据库规则
- 认证规则
- 代码风格
```

### 3. 文件结构
```markdown
## File Structure
项目的目录结构
```

### 4. 核心模式
```markdown
## Key Patterns
- API 响应格式
- 错误处理
- 服务层模式
```

### 5. 环境变量
```markdown
## Environment Variables
需要的环境变量列表
```

### 6. 测试策略
```markdown
## Testing Strategy
测试命令和覆盖率要求
```

### 7. ECC 工作流
```markdown
## ECC Workflow
可用的斜杠命令
```

## 用户级 vs 项目级配置

| 配置类型 | 位置 | 作用范围 |
|----------|------|----------|
| 用户级 | `~/.claude/CLAUDE.md` | 所有项目 |
| 项目级 | `项目根目录/CLAUDE.md` | 单个项目 |

**推荐做法**：
- 用户级配置：放个人偏好、通用规则
- 项目级配置：放项目特定规则、技术栈配置
