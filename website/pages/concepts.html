00A：概念扫盲（必看）

预计时间：15分钟 难度：超简单 类别：必修
Claude Code 工作流程：从你的输入到AI响应
为什么要看这页？

新手最容易卡在“名词看不懂”：模型、Key、命令行、目录、权限……

这页用大白话把它们的关系讲清楚，看完你就不慌了。

一张图先看懂（建议收藏）

你输入的“中文需求” │ ▼ Claude Code CLI（你电脑上的工具） │ 负责：读文件/写文件/跑命令/展示对话 │ ├─ 需要：权限（只读/默认/危险/系统级） │ └─ 连接到：模型服务（真正回答你） │ ├─ 需要：API Key（像“门禁卡/密码”） └─ 可能收费：按用量计费（别乱试太多）

核心名词（你只要记住这几句）

名词 你可以这样理解 常见误解
Claude Code CLI 一个“会干活的命令行助手”（在你电脑上运行） 不是网页聊天；它能读写你的文件
模型 真正“回答你/写代码”的大脑（在云端或本地） 不是你电脑里的程序
API Key 连接模型服务的“门禁卡/密码” 发给别人=可能被刷额度
工作目录 你让 Claude Code “在哪个文件夹里干活” 不是系统盘根目录；别随便给全盘权限
权限模式 你允许它“能做什么” 危险模式能删文件，慎用

新手最安全的默认用法（强烈推荐）

  1. 只在你自己的项目文件夹里用（不要全盘）。
  2. 先用默认模式/只读模式熟悉流程。
  3. 任何“批量删除/批量移动”，先 Git 存档或备份。

隐私与脱敏（强烈建议）

  • API Key 只放在环境变量/配置文件里，不要发聊天记录、不要截图发群。
  • 公司/客户资料 先脱敏：姓名/电话/身份证/地址/合同金额等用占位符替换。
  • 最小权限：只给 Claude Code 你希望它处理的那个文件夹，不要全盘授权。
